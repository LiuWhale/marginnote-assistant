# MarginNote 自带 AI 对话对标矩阵

## 来源

- https://forum.marginnote.com/t/questions-about-the-new-ai-assistant/11047

- https://www.marginnote.com/en/

## 产品边界

Codex Companion 当前只对标 MarginNote 自带 AI 的对话体验：用户在阅读时把上下文交给 AI，然后用一个输入框连续问答。非目标：制卡、脑图生成、原文高亮、导出 PDF、文件上传、发布验收、队列管理、自定义按钮中心。这些能力可以继续由 Companion 后台保留给维护和实验使用，但不作为主界面可见入口。

## 对标矩阵

| 能力 | MN4 自带 AI 依据 | Codex Companion 当前策略 | 状态 |
| --- | --- | --- | --- |
| 显式上下文授权 | 官方论坛说明 AI 只访问用户明确发送的内容，例如拖入卡片、PDF 选区、文本、模板或工具输出。 | 主界面固定显示 `AI 可见：...`，只把 MN4 当前传入的上下文和用户输入发送给 Companion。 | 已对齐 |
| 选区 / 节点 / 文档上下文 | MN4 AI 的核心使用方式是围绕阅读材料、卡片和选区进行问答。 | 上下文来源行区分 `PDF 选区`、`脑图节点`、`当前文档`，并在预览区显示实际可见内容。 | 已对齐 |
| 多模型档位 | 官方论坛列出 Balanced、Lightweight、Text、Depth、Visual、Tool 等模型类型。 | 插件后台支持 Codex CLI / OpenAI API / 本地诊断后端；最小对话界面不暴露复杂模型档位，避免把设置变成主流程。 | 部分对齐 |
| 生成中停止 | AI 对话需要明确的中断入口，官方更新也持续改善 AI/Research 浮窗交互。 | 运行中显示 `停止` 按钮，调用 `stop_current`，按钮不替代发送入口，也不暴露队列控制。 | 已对齐 |
| 对话式阅读辅助 | 官网定位是阅读助手：可总结章节、提示关键术语、生成问题，用户可以接受、编辑或拒绝建议。 | 默认只保留聊天输入框和历史消息，回答以阅读问答为中心，不再自动弹后续制卡/脑图引导。 | 已对齐 |

## 非目标

- 不把“目标”“一次性任务”“队列”“按钮中心”放进主界面。
- 不在聊天后自动推荐制卡、脑图、高亮、导出等动作。
- 不承诺复制 MarginNote 官方 Visual/Tool 模型的全部能力；当前只对齐 AI 对话体验和显式上下文交互。
- 不让插件后台自动扫描整个库；上下文来自 MN4 当前传入内容、当前文档标识和用户输入。
