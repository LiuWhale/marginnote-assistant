# 工具中心与专家模式使用手册

适用版本：`0.4.41`

`工具` 是 Codex Companion 的可选任务中心。默认首屏仍是 `对话`，用于快速解释选区、连续追问和普通问答；当你想精读资料、生成脑图、生成复习卡，或检查写入/回滚时，再切到 `工具`。工具页默认只显示当前状态、四个大任务按钮和下一步建议。

旧的对象、来源、脑图 Diff、制卡质量、workflow、验证、回滚和排错面板没有删除，但默认收在 `专家模式` 里。只有当你需要看证据、账本、对象路线、workflow 或排错细节时，才需要展开专家模式。

## 1. 什么时候用工具中心

用 `对话`：

- 快速解释选区。
- 连续追问概念。
- 只想像 MarginNote 自带 AI 那样问一句、答一句。

用 `工具`：

- 精读当前文档。
- 生成或重组脑图。
- 把回答补到当前脑图。
- 生成复习卡并检查卡片覆盖。
- 查看失败原因、验证证据或回滚事务。
- 需要确认当前插件到底看到了哪些 PDF、选区、节点和 MN 对象。

## 2. 顶部模式

顶部有两个模式：

- `对话`：默认模式，展开完整对话历史，适合轻量问答。
- `工具`：可选模式，默认显示任务中心，用于解读资料、生成/整理脑图、生成复习卡、检查写入/回滚。

切到工具中心后，底部仍然保留输入框和 `发送 / 可排队`。如果当前有任务在运行，新请求会进入 pending 队列，等上一个任务结束后继续执行。

## 3. 工具任务中心

工具页默认有三块：

| 区域 | 作用 |
| --- | --- |
| `当前状态` | 显示材料是否可读、目标脑图是否明确、最近写入是否需要验证或回滚 |
| `下一步建议` | 根据当前状态推荐最优先动作，例如选择 PDF、刷新目标脑图或检查写入 |
| `你想做什么` | 四个大按钮：解读当前资料、生成/整理脑图、生成复习卡、检查写入/回滚 |

前三个生成类任务不会直接跑长任务。点击后会把合适的 prompt 放进对话输入框，你可以先改文字，再点发送或按 Enter；这样不会因为误触按钮就开始长时间生成。

## 4. 专家模式导航

专家模式会显示旧的 Workspace Navigator 和完整工作台。现在主要通过下拉菜单切换页面：

| 页面 | 作用 |
| --- | --- |
| `Knowledge Console` | 当前 notebook 总览，检查来源、对象、脑图、卡片、workflow 和账本证据 |
| `Mindmap Studio` | 读取当前脑图、预览 Diff、应用所选、验证或回滚事务 |
| `Card Factory` | 生成、检查和管理复习卡 |
| `Operation Ledger` | 查看执行记录、失败原因、证据和回滚残留 |
| `Knowledge Graph` | 查看对象、关系和检索结果 |
| `Workflow Builder` | 查看和启动长任务 workflow |
| `Skill Center` | 查看技能包、权限和可运行能力 |

普通用户不需要频繁进入这里。专家模式用于诊断、验收和对象级操作，不作为默认学习入口。

## 5. Notebook Workspace

Notebook Workspace 是专家模式里的总览。它先回答三个问题：

1. 当前对象是什么：文档、选区、节点、PDF、脑图或 notebook。
2. 当前材料是否可读：PDF 缓存、上传文件、文件路径和 MarginNote 上下文是否可用。
3. 当前能安全做什么：读材料、看 MN 对象、整理脑图、做复习卡、跑 workflow、用技能包或验证回滚。

颜色含义：

- 绿色：已通过，可以直接使用。
- 黄色：需要操作或等待。
- 红色：阻断，需要补权限、上下文、PDF 或目标脑图。
- 灰色：暂时没有数据。

## 5. Object Intake

Object Intake 会把当前文档、选区或节点路由到合适工作区。

常见路线：

- `读材料`：进入 Source Registry，确认 PDF、缓存、上传文件是否可读。
- `看 MN 对象`：进入 Object Browser，查看当前对象和相关 MN 原生对象。
- `整理脑图`：进入 Mindmap Studio。
- `做复习卡`：进入 Card Factory。
- `跑长任务`：进入 Workflow Builder。
- `用技能包`：进入 Skill Center。
- `验证/回滚`：进入 Verification Center 或 Operation Ledger。

如果某条路线显示阻断，不要直接强行生成。先点对应按钮补齐前置条件。

## 6. Mindmap Studio

Mindmap Studio 是脑图对象操作台，不是“生成脑图按钮”的别名。

推荐流程：

1. 在顶部确认目标脑图。
2. 点 `读取当前脑图`，拿到现有脑图基线。
3. 在聊天框输入需求，例如“把刚才回答补到当前脑图”。
4. 点 `预览 Diff`，查看新增、更新、移动、合并和建议删除。
5. 修改或取消不想写入的节点。
6. 点 `应用所选`。
7. 写入后点 `验证事务`。
8. 如果不满意，点 `回滚事务` 或在 AI 编辑确认里点拒绝。

原则：先 Diff，后写入；写入后必须可验证、可回滚。

## 7. Card Factory

Card Factory 用来生成和检查复习卡。

推荐流程：

1. 先确认当前对象或选区正确。
2. 输入“根据当前选区生成复习卡”或从 Object Intake 进入制卡路线。
3. 查看卡片草稿。
4. 检查卡片是否过长、是否缺来源、是否重复。
5. 确认后写入 MarginNote。
6. 需要复习闭环时，加入复习队列。

卡片应该短而明确，不应该把一大段解释塞进一张卡。

## 8. Operation Ledger 和 Verification Center

这两个区域用于排错和验收。

用 `Operation Ledger` 看：

- 最近执行了什么。
- 哪一步失败。
- 失败时的原始错误。
- 是否生成了事务。
- 回滚后是否有残留。

用 `Verification Center` 看：

- 当前对象是 `PASS`、`FAIL` 还是 `UNKNOWN`。
- 是否缺真实 MN 对象 probe。
- 是否缺 PDF 来源。
- 是否需要执行推荐修复。

`UNKNOWN` 不等于成功，也不等于失败。它表示证据不足，不能把日志成功当成真实 MarginNote 成功。

## 9. Workflow Builder

Workflow Builder 负责长任务。

典型任务：

- 文档精读 workflow。
- 当前脑图重组 workflow。
- 选区制卡 workflow。
- 自动准备 notebook 工作台。

workflow 里的生成类步骤可以排队执行；写入类步骤必须停在确认点，不能绕过 Diff、接受/拒绝和账本验证。

## 10. Command Pane

底部 Command Pane 是输入入口。

- 收起时：只显示输入框和发送按钮，不挡工作台。
- 展开时：显示完整对话历史。
- 展开对话时，Workspace Navigator 会自动收起，避免挤压。
- `发送 / 可排队` 表示：空闲时立即执行，忙时进入 pending 队列。

常用输入：

```text
解释当前选中内容
```

```text
精读这篇文章，并给出结构化摘要
```

```text
把上面的回答补到当前脑图里
```

```text
根据当前选区生成短复习卡
```

```text
检查刚才写入的脑图事务有没有残留
```

## 11. 推荐工作流

### 精读当前文档

1. 进入 `高级`。
2. 在 `Knowledge Console` 看 PDF 缓存是否为绿色。
3. 如果不是绿色，点 `选择 PDF` 或缓存当前 PDF。
4. 输入“精读这篇文章”。
5. 需要脑图时，点回答下方的 `生成脑图树`。
6. 到 `Mindmap Studio` 预览 Diff。
7. 应用所选并验证事务。

### 补到当前脑图

1. 在 MarginNote 中确认目标脑图或目标节点。
2. 在插件顶部确认目标脑图名称。
3. 输入“把刚才内容补到当前脑图”。
4. 进入 `Mindmap Studio`。
5. 预览 Diff。
6. 应用所选。
7. 验证事务。

### 排查失败

1. 打开 `Operation Ledger`。
2. 查看最近失败的 ledger。
3. 如果是权限问题，进入设置页检查权限。
4. 如果是 PDF 问题，回到 Source Registry 补来源。
5. 如果是脑图写入问题，进入 Mindmap Studio 验证或回滚。

## 12. 使用原则

- 不确认目标脑图，不写入脑图。
- 没有 PDF 来源，不做全文精读。
- 没有 Diff，不直接应用脑图变更。
- 没有验证，不把写入当成成功。
- 长任务进 workflow，短问题用 `对话`。
- 失败先看 Operation Ledger，不要反复点同一个按钮。
